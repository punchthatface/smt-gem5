Phase 2 handoff: final architecture-aware SMT experiment
=========================================================

Final target:
  4 cores, 2 SMT threads/core = 8 hardware contexts.
  8 unmodified PARSEC workloads.
  4 policies:
    1. no_smt
    2. unrestricted_smt
    3. constrained_smt
    4. flush_smt

Current implementation:
  configs/smt_domain_flush.py
    - gem5 SE config
    - supports periodic L1D cleanup with:
        dcache.memWriteback()
        dcache.memInvalidate()
    - supports SMT resource policy forwarding:
        --smt-resource-policy partitioned

  scripts/run_phase2_experiment.py
    - main runner
    - forwards flush and SMT resource-policy fields from JSON to gem5
    - writes CSV stats

SMT resource policy status:
  Tested on node0.
  Applied:
    smtFetchPolicy = RoundRobin
    smtCommitPolicy = RoundRobin
    smtROBPolicy = Partitioned
    smtLSQPolicy = Partitioned
  Not supported:
    smtIQPolicy = Partitioned

8 workload set:
  swaptions       gcc-pthreads
  blackscholes    gcc-pthreads
  streamcluster   gcc-pthreads
  fluidanimate    gcc-pthreads
  canneal         gcc-pthreads
  freqmine        gcc-openmp, OMP_NUM_THREADS=1
  bodytrack       gcc-pthreads
  dedup           gcc-pthreads

Setup helper:
  scripts/setup-phase2-parsec.sh

Optional run-dir copy helper:
  scripts/copy-phase2-parsec-runs.sh

Directory policy:
  scripts/phase2_configs/test/   short tests only
  scripts/phase2_configs/final/  final configs only
  results/test_runs/             short tests only
  results/final/                 final runs only

Important:
  Pairwise 1-core/2-thread configs are not the final-final experiment.
  They are only sanity/evaluation extras.
  Final-final must use the 4-core/8-thread policy mix.
